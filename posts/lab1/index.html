Setting up the blog locally Link to heading

Hugo brief description Link to heading

Hugo is a fast, open-source static site generator designed for building websites quickly. Written in Go, it takes content in Markdown files and templates to generate static HTML files, making it ideal for blogs, documentation, and portfolios. Hugo supports extensive customization through themes, templates, and configurations, and it allows for easy deployment to various platforms, including GitHub Pages and Netlify. With its speed and flexibility, Hugo is popular among developers and content creators who want a lightweight, scalable website solution.

Installing hugo and building the site Link to heading

Prerequisites: having installed git Hugo can be installed with this simple command:


sudo snap install hugo

To create new site, this command must be written. You cna replace “BlogdeKanita” by the name you want to give to your site.


hugo new site BlogdeKanita
You then have a site at one of your ports of the localhost. The cotnent of the site is inside the new folder with your blog'sname. Be sure to be in this folder before doing the other steps. 

cd ./BlogdeKanita/

Setting up the blog theme Link to heading

Prerequisites: having git installed, being in the site foder. You have to chose a theme among those available. I choosed the hugo-coder one. Then, follow the instructions to dowload the theme. The command has this form:


git submodule add https://github.com/luizdepra/hugo-coder.git themes/hugo-coder

where the adress is the one of the git where the theme you want to download is.

Set the theme in config.toml:


theme = "hugo-coder"

You now have a site that is working locally with a nice theme.

Personnalise the site Link to heading

You can not edit the config.toml file to personnalize the site to your liking. To test your site locally, run:


hugo server -D
You can now see your site at the adress http://localhost:1313.

Deploying the blog globally Link to heading

Here’s how I set up the game globally using github pages.

You have to create a new github repository. Mine is called KanitaBlog. Now, you can push your project on github:


git status       
git add .        # Add all files
git commit -m "Initial commit" 
git branch -M main  # creating the main branch 
git remote add origin https://github.com/Kani12345/KanitaBlog.git  # https://yourgithubusername.com/yourrepositoryname.git
git push -u origin main

Now, you have to configurate your repository and your site by following hugo’s tutorial on deploying a blog on github pages.

Congrats, you now have your own blog!
